# Getting help with Tripwire

A few pointers so your question lands in the right place.

## Setup and usage

Most setup problems are covered in the README:

- Install for your MCP client, plus the prebuilt-binary and from-source alternatives.
- The Studio plugin install, and enabling Allow HTTP Requests (the most common setup failure).
- The full Open Cloud key walkthrough: scopes, universe and place ids, and the `.env`.
- The Troubleshooting section, which lists the common first-run failures and their fixes.

Read those first. If a tool returns a Roblox error, it is usually a missing Open Cloud scope; the README lists which scope each tool needs.

## Asking a question

If the README does not cover it, open an issue with the bug report or feature request template. Include your MCP client, your OS, the Tripwire server version, and whether the tool used the Studio path or the Open Cloud path.

## Bugs and features

- Bug reports and feature requests go in the issue tracker. See [CONTRIBUTING.md](CONTRIBUTING.md).
- Security issues do not. See [SECURITY.md](SECURITY.md) for the private path.

This is a small project maintained in spare time, so replies may take a while. A clear, reproducible report gets a faster answer.
